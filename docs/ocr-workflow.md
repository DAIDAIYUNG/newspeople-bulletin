# 新聞人報社社訊 — OCR 與掃描技術維護文件

本文件記錄社訊掃描、OCR、校對與上架流程，作為後續維護與擴充的標準作業程序（SOP）。

---

## 1. 掃描來源與檔案格式

### 1.1 掃描來源
- 原始紙本社訊（2005–2010）
- 來源品質不一，部分紙張泛黃、折痕、墨跡不均

### 1.2 掃描規格（建議）
- **解析度：300 dpi**
- **色彩模式：灰階（Grayscale）**
- **檔案格式：PNG 或 TIFF（避免 JPEG 壓縮）**
- 每一期建立一個資料夾，例如：

images/ └── 2005/ ├── 0406/ ├── 0608/ └── 1104/

---

## 2. OCR 處理流程

### 2.1 使用工具
建議使用下列任一 OCR 工具：

- **Google Drive OCR（最穩定）**
- **Adobe Acrobat OCR**
- **Tesseract OCR（開源）**

### 2.2 OCR 步驟（以 Google Drive 為例）
1. 將掃描圖片上傳至 Google Drive
2. 右鍵 →「以 Google 文件開啟」
3. Google 會自動執行 OCR
4. 複製辨識後的文字至 Markdown 檔案

---

## 3. OCR 常見問題與修正方式

### 3.1 文字錯誤
常見錯誤：
- 「問」辨識成「間」
- 「報」辨識成「報」
- 標點符號遺失
- 段落斷行錯誤

**修正方式：**
- 使用 VS Code 的「取代」功能快速修正
- 逐段比對掃描圖片與 OCR 文字
- 保留原文格式，不任意改寫內容

---

## 4. Markdown 文章格式規範

每篇文章需建立一個 Markdown 檔案，放在 `_posts/`：

### 4.1 Front Matter 格式

```yaml
---
layout: post
title: "文章標題"
date: 2005-04-06
categories: 社訊
issue: "2005-0406"
---

4.2 內文格式
使用標準 Markdown
圖片引用格式：
 ![描述文字](/images/2005/0406/2005-0406-01.png)



5. 圖片命名規範
5.1 命名格式
YYYY-期數-頁碼.png

例如：
2005-0406-01.png
2005-0406-02.png

5.2 資料夾結構
images/
 └── 2005/
      ├── 0406/
      │    ├── 2005-0406-01.png
      │    ├── 2005-0406-02.png
      └── 0608/
           ├── 2005-0608-01.png


6. 上架流程（Jekyll）
6.1 新增文章
將 Markdown 檔案放入：
_posts/

檔名格式：
YYYY-MM-DD-title.md

6.2 新增年份頁
在 years/2005.html 中會自動顯示該年份所有文章。

7. Git 推送流程
git add .
git commit -m "Add OCR text for 2005-0406 issue"
git push


8. 後續維護建議
每新增一期社訊，請依照本文件流程處理
建議建立 CHANGELOG.md 記錄每次新增的期數
若 OCR 品質不佳，可考慮重新掃描或使用更高品質 OCR 工具
若未來要加入搜尋功能，可整合 Lunr.js 或 Algolia

本文件將持續更新，以確保社訊資料能長期保存並維持良好品質。
